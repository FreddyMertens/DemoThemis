Mechanism review · 23 July 2026

Replace fixed jury tiers with adaptive security selection

Review of: case value + eligible pool + attack assumptions → test configurations → choose the smallest safe ruling set.

✓
Recommended—with strict guardrails.

Adaptive selection is materially better than fixed dollar tiers, but only as a deterministic protocol rule over a small published configuration set. A requester-controlled slider or a single optimistic attacker estimate would be worse.

Why fixed tiers are weak

They create arbitrary cliffs, ignore changes in the drawable cohort, overpay for some cases, underprotect others, and reward understating or splitting the value affected.

Why a hybrid selector is better

It preserves predictable legal options while choosing from actual security, quality and capacity conditions. The result is reproducible without pretending risk can be estimated perfectly.

Recommended mechanism

Select the lowest-cost configuration that passes every limit

Keep five versioned candidates: 7, 15, 31, 3×31 and 5×31. For parallel sets, the review recommends testing 2-of-3 and 3-of-5 panel majorities instead of requiring unanimity.

Freeze economic exposureBuild candidate-specific poolStress every configurationChoose first safe option

For candidate C, use:

  • V: maximum economically linked value the ruling can redirect, taken from locked backing—not user-entered text.
  • NC: matured, available, qualified and conflict-free jurors after parties and all earlier jurors are excluded.
  • Θ: a published conservative envelope for attacker selection weight, rentals, no-shows and correlated errors.
  • Pcontrol: exact finite-pool probability under disjoint draws and the real cross-panel decision rule.
riskLimit(V) = min(1%, Lmax / V) Pass(C) only if: max over Θ Pcontrol(C) ≤ riskLimit(V) honest-resolution probability ≥ Qmin quorum probability ≥ Fmin NC ≥ required seats + standbys deterministic fee is fundable Choose the least-cost passing C; otherwise SAFETY_HALT.

Lmax is a published maximum expected corrupted-value budget. The existing 1% Break the Court benchmark can remain the probability ceiling, but Lmax, Qmin, Fmin and the standby buffer require explicit calibration.

Necessary protections

Prevent the selector itself becoming the attack

  • Freeze V, the eligible-set root, exclusions, policy version and chosen configuration before future randomness exists.
  • Use upper attacker-weight and lower pool estimates from a lagged epoch—not live sliders or requester claims.
  • Recompute eligibility, workload, retention and fees separately for every candidate; those values change with panel size.
  • Use exact without-replacement parallel math. The current single-panel probability raised to a power is not exact.
  • Aggregate linked markets and escrows so one exposure cannot be split into several apparently small cases.
  • If nothing passes, reject or queue the case and return unused funding. Never fall back to an unsafe maximum panel.

Further system improvements

The review exposes three adjacent design upgrades

  1. Fix parallel-panel finality. Unanimity lets one disrupted panel stall the ruling and makes honest disagreement increasingly likely. Model majority-of-panels plus escalation-on-split, and publish both capture and no-verdict probabilities.
  2. Extend appeals. Use one strict order: 7 → 15 → 31 → 3×31 → 5×31, always with unused jurors. This gives top-tier disagreement a real next rung; 5×31 is terminal.
  3. Improve Break the Court. Default to Automatic, with 7/15/31/3×31/5×31 as clearly experimental overrides. Show the chosen set, worst-case control probability, expected loss, honest-finality probability and spare eligible seats.

Important: a larger panel cannot repair a cohort in which attacker selection weight approaches 50%, nor can several panels correct a shared evidence failure. Pool health and evidence quality remain separate admission checks.

Final recommendation

Prototype this in Break the Court before replacing the published schedule

First validate monotonicity, exact parallel math, candidate-specific pool equilibrium, invalid-value failure, case-splitting resistance, appeal progression and no-configuration halts. Once calibrated, DemoThemis should publish the selector and show fixed tiers only as illustrative outputs under reference assumptions—not as permanent law.

Scope: complete-product mechanism design. This review proposes no change to the live three-seat demo MVP.
