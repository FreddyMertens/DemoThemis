Game lab audit explainer

The problem is not the protocol. The problem is overclaiming what the lab proves.

This page translates the audit into plain English. The game board is useful, but it should be presented as an illustrative stress model, not as a calibrated proof that every attack has been solved.

Short verdict

The demothemis design is not exposed as fundamentally broken by the game page. The risky part is that the page makes teaching shortcuts look like hard science.

The safest framing: “This lab shows dependencies and pressure points. It is not a launch forecast, a parameter vote, or a measured probability model.”

The issues

These are ranked by credibility risk: what could make a reader think the page is proving more than it really proves.

Highest risk

“No magic constants” is not true

The model has hidden assumptions: safety weights, watcher curves, rented-seat pass rates, whale size, appeal values, and key-signal scaling.

Plain English: some numbers are chosen by the model author, not forced by the protocol.

Misleading label

“Protocol safety” sounds objective

The safety number is a weighted score. It is not a measured probability that the protocol will work.

Fix: call it a “stress score” or “illustrative safety index.”

Hidden floor

Reserve does not always scale smoothly

The copy says reserve rises with unresolved workload. The model also uses a one-pending-vote floor, so it can stay high even when workload is tiny.

Fix: disclose the floor or make it visible.

Too precise

Watcher catch odds look empirical

The catch percentage comes from a smooth formula, not from real watcher data.

Fix: call it “watcher coverage” unless it is calibrated from evidence.

Proxy model

Governance is simplified

The governance chapter has richer rules: verified identity, service multipliers, token locks, and caps. The lab uses a simpler whale-versus-users proxy.

Fix: either implement the fuller model or label this as a proxy.

Approximation

Panel capture is approximate

The lab uses binomial sampling. Real panel draws from a finite juror pool are closer to drawing without replacement.

Fix: use finite-pool math or say the displayed odds are approximate.

Completeness

Very large cases omit a defense

The court design includes parallel panels above the panel-size ceiling. The lab currently stops at a single large panel.

Fix: add the multi-panel ceiling defense or narrow the claim that this is the “whole protocol.”

Mostly safe

Privacy framing is close

The page mentions consented private tests, which is good. But some lines still make “settled traffic grades jurors” sound too broad.

Fix: say public outcomes and minimized, consented private tests feed scoring.

Not fatal

The protocol is not disproven

The audit does not show that demothemis fails. It shows that the explainer page should be more honest about its assumptions.

Fix: present the board as a dependency map, not a proof.

Safer wording

These changes preserve the message without sounding like the lab proves more than it can prove.

Avoid: “There are no magic constants here.”
Use: “Every dial has a job, and the assumptions are printed where they matter.”
Avoid: “Protocol safety.”
Use: “Illustrative safety index” or “stress score.”
Avoid: “Quiet settlements feed scoring.”
Use: “Public outcomes and minimized, consented private tests feed scoring.”
Avoid: “The whole protocol as a game board.”
Use: “A simplified game board for the protocol’s main incentive loops.”

Fix order

This is the fastest path to making the game page credible without throwing away the useful model.

Rename the overconfident parts

Change the “no magic constants” heading and the “Protocol safety” label first. This removes the biggest credibility problem immediately.

Add an assumptions panel

Show the hidden constants: rented-seat pass rate, watcher curve, key-signal curve, whale token assumption, grief value, delay value, and safety weights.

Make reserve and governance assumptions explicit

Explain the reserve floor. Either align governance with the governance chapter or mark it clearly as a simplified proxy.

Decide whether this is “whole protocol” or “simplified model”

If it remains “whole protocol,” add the multi-panel ceiling defense. If not, narrow the language.

Tighten privacy and escrow framing

Say the canonical model is protocol-held escrow plus private-by-default arbitration, with minimized scoring signals and signed verdict receipts.

Bottom line

The game lab is a strong explainer if it is framed as a dependency map. It becomes risky when it sounds like a proof.

Make the assumptions visible, rename the headline score, disclose the simplified math, and tighten privacy wording. Then the page teaches the protocol without overstating what the model can prove.
